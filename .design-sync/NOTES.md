# design-sync notes

- **Repo shape:** `mobile/` is a React Native / Expo app, NOT a DOM component
  library. Components in `mobile/src/components/` build on react-native
  primitives (`View`, `Text`, `Pressable`, `Animated`) and native Expo modules
  (`expo-blur`, `expo-linear-gradient`, `@expo/vector-icons`, `@zxing/browser`),
  and depend on app providers (`AuthContext`, `ThemeContext`). They cannot render
  in claude.ai/design. There is no Storybook and no `*.stories.*`.
- **Scope decision (2026-06-13):** user chose a **tokens-only** sync. Bundle is
  generated by hand from `mobile/src/lib/theme.ts` into `ds-bundle/` — no
  `_ds_bundle.js`, no components.
- **Re-sync:** regenerate `ds-bundle/tokens/*` + `styles.css` from
  `mobile/src/lib/theme.ts` whenever the palette/scale/gradients change. Light is
  the app default (`:root`); dark is `[data-theme="dark"]` / `.dark`.
- To add real components later, you'd need a DOM/web component library (e.g.
  extracted via react-native-web into its own bundlable package). Not in scope now.
